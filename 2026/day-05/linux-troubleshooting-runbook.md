# Day 05 – Linux Troubleshooting Practice

## Task

Today I practiced some basic Linux troubleshooting commands.

I selected one running service on my system and checked its health by looking at CPU, memory, disk usage, network connections, and logs.

I also wrote a small runbook so I know what to check first if the service has any problem in the future.

---

## What is a Runbook?

A runbook is a simple guide that contains the commands I should run during a problem and the steps to fix it.

It helps me troubleshoot issues quickly without forgetting important checks.

---

## My Goal

By the end of today's practice, I will create:

- `linux-troubleshooting-runbook.md`

or

- A handwritten troubleshooting note.

---

## What I Will Practice

### 1. System Information
- Check Linux version.
- Check OS details.

### 2. Files and Directories
- Create a temporary folder.
- Create or copy a file.
- Verify everything is created correctly.

### 3. CPU and Memory
- Check running processes.
- Check memory usage.

### 4. Disk Usage
- Check available disk space.
- Check log folder size.

### 5. Network
- Check listening ports.
- Test network connectivity.

### 6. Logs
- View logs of my selected service.
- Read the latest log entries.

### 7. My Findings
For every command, I will write:
- What I checked.
- What I observed.
- Whether everything looks normal.

### 8. If the Problem Gets Worse

If the issue continues, I would:

- Restart the service.
- Check more detailed logs.
- Collect more information before making changes.

---

## Why I Am Learning This

As a DevOps Engineer, troubleshooting is an important skill.

Instead of guessing the problem, I should first collect information using Linux commands.

This practice helps me become faster and more confident while handling real production issues.
